# 03_API_INTEGRATION_PLAN — 공공 API 연동 계획

> 방침: 공공 API 연동은 **가산점**이며 **필수가 아니다**. **모든 API는 실패 시 샘플 데이터 모드로 자동 전환**되어야 하고, 데모 흐름은 절대 끊기지 않는다.
> 모든 외부 호출은 **Next.js Server Route(서버)에서만** 수행한다(키 노출 방지, CORS 회피, 캐시·타임아웃 일원화).
> 확정 결정: **가능한 많이 연동**하되, 우선순위에 따라 단계적으로 붙이고 fallback을 먼저 보장한다.

---

## 1. 공통 연동 원칙

- **위치**: `app/api/external/*` Server Route. 클라이언트는 내부 API만 호출.
- **fallback 자동화**: 각 외부 호출은 `try → 실패/타임아웃/키없음 → 샘플 응답 반환` 구조. 응답에 `source: 'api' | 'sample'` 메타 포함.
- **강제 토글**: `USE_SAMPLE_FALLBACK=true`면 외부 호출을 시도하지 않고 즉시 샘플 반환(오프라인 데모).
- **타임아웃/재시도**: 기본 timeout 5s, 1회 재시도 후 fallback.
- **캐시**: 동일 좌표/지역 단기예보 등은 메모리/Route 캐시(수 분) 적용해 호출 절감.
- **개인정보**: 외부 API에 개인식별정보 전송 금지(좌표·지역코드 등 비식별 정보만).

---

## 2. 연동 후보 API 목록

| # | API | 역할 | 난이도 | MVP 우선순위 | fallback |
|---|---|---|---|---|---|
| 1 | **주소검색·좌표변환** (도로명주소/카카오 로컬 등) | 기관등록 시 주소→위경도·행정구역 자동 채움 | 낮음 | **高** | 위경도 수기 입력 + 샘플 주소 좌표 매핑 |
| 2 | **기상청 단기예보 조회서비스** | 기관 좌표 기준 현재/단기 기온 등 → AI 근거 보강 | 중 | **高** | 샘플 기상값(예: 체감 35℃) 주입 |
| 3 | **기상청 영향예보(폭염) 조회서비스** | 폭염 영향 등급/위험수준 → 우선순위 산정 근거 | 중 | 中 | 샘플 영향등급(주의/경고) |
| 4 | **행정안전부 긴급재난문자 API** | 지역 실제 폭염 재난문자 자동 수집 | 중~상 | 中 | 샘플 재난문자 3종(02·04 공유) |
| 5 | **한국사회보장정보원 어린이집 정보 API** | 어린이집 기관 정보 자동 입력 | 상 | 低 | 수기 입력 + 샘플 기관 |
| 6 | **한국교육학술정보원 유치원 공시정보 API** | 유치원 기관 정보 자동 입력 | 상 | 低 | 수기 입력 + 샘플 기관 |

> 연동 순서 권장: **1 → 2 → 3 → 4 → (5,6)**. 1·2가 데모 체감 효과 대비 난이도가 낮다. 5·6은 공공데이터포털 신청·승인 지연이 커서 후순위.

## 3. API별 상세

### 3.1 주소검색·좌표변환 (우선순위 高)
- **역할**: 기관등록 폼에서 주소 입력 → 위도/경도/시도/시군구/행정동 자동 채움.
- **후보**: 카카오 로컬 REST(주소→좌표) 또는 행안부 도로명주소 API + 좌표변환. MVP는 키 발급이 빠른 쪽 1개 선택.
- **엔드포인트(예시)**: `GET /api/external/geocode?query=...` → `{ lat, lng, sido, sigungu, dong, source }`.
- **fallback**: 시드에 포함된 샘플 주소→좌표 테이블에서 근사 매핑, 없으면 사용자 수기 입력 유지.

### 3.2 기상청 단기예보 (우선순위 高)
- **역할**: 기관 좌표(격자 변환) 기준 기온/체감온도 등을 AI 입력의 `weather_context`로 제공 → "반영된 근거 정보"에 실데이터 표기.
- **엔드포인트(예시)**: `GET /api/external/weather/short-term?lat=&lng=` → `{ temp, feelsLike, time, source }`.
- **주의**: 기상청 격자(nx, ny) 변환 로직 필요(위경도→격자). 변환 유틸 별도.
- **fallback**: 샘플 기상값 주입(예: 기온 34℃, 체감 36℃).

### 3.3 기상청 영향예보(폭염) (우선순위 中)
- **역할**: 폭염 영향 등급(관심/주의/경고/위험 등) → `priority` 산정 근거 보강.
- **fallback**: 샘플 영향등급.

### 3.4 행안부 긴급재난문자 API (우선순위 中)
- **역할**: 지역 폭염 재난문자 자동 조회 → 재난문자 입력 단계에서 "최근 문자 불러오기" 옵션.
- **fallback**: 샘플 재난문자 3종(붙여넣기/샘플 선택 흐름은 항상 동작).

### 3.5 / 3.6 어린이집·유치원 정보 API (우선순위 低)
- **역할**: 기관명/유형/주소/정원 등 자동 입력으로 등록 편의.
- **현실 제약**: 신청·승인·키 발급 지연 큼 → MVP 데모는 수기 입력 + 샘플 기관으로 충분. 시간 여유 시 추가.
- **fallback**: 수기 입력 폼 + 시드 기관.

## 4. fallback 전략 요약

```
호출 흐름:
  USE_SAMPLE_FALLBACK? ──yes──> 샘플 응답 (source:'sample')
        │ no
        ▼
  키 존재? ──no──> 샘플 응답
        │ yes
        ▼
  외부 호출 (timeout 5s) ──실패/타임아웃──> 1회 재시도 ──실패──> 샘플 응답
        │ 성공
        ▼
  실데이터 응답 (source:'api')
```
- UI는 `source` 메타로 "실데이터/샘플" 배지를 표시할 수 있다(투명성 + 데모 설명).
- **핵심 흐름(AI 생성·결과·사후기록)은 어떤 외부 API 없이도 100% 동작**해야 한다.

## 5. 환경변수 목록 (`.env.local` / Vercel 환경변수)

| 변수 | 용도 | 비고 |
|---|---|---|
| `ANTHROPIC_API_KEY` | Claude API | 서버 전용 |
| `ANTHROPIC_MODEL` | 모델명 | 기본 `claude-haiku-4-5`(고품질 시 `claude-sonnet-4-6`) |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase URL | 클라이언트 노출 가능 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon 키 | 클라이언트 노출 가능 |
| `SUPABASE_SERVICE_ROLE_KEY` | 서버 라우트 쓰기/민감 조회 | **절대 클라이언트 노출 금지** |
| `USE_SAMPLE_FALLBACK` | 외부 의존 전면 차단 토글 | `true`/`false` (데모 안전판) |
| `GEOCODE_API_KEY` | 주소·좌표 변환 | 서버 전용 |
| `KMA_API_KEY` | 기상청(단기예보/영향예보) | 서버 전용 |
| `MOIS_DISASTER_API_KEY` | 행안부 긴급재난문자 | 서버 전용 |
| `CHILDCARE_API_KEY` | 어린이집 정보(선택) | 서버 전용 |
| `KINDERGARTEN_API_KEY` | 유치원 공시(선택) | 서버 전용 |

> 모든 외부 키는 **미설정 시 자동 샘플 모드**로 동작하도록 코드에서 분기한다(키 없어도 빌드·데모 가능).

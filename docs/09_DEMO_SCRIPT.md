# 09_DEMO_SCRIPT — 3분 시연 스크립트

> 공모전 심사 현장에서 **3분 내 핵심 가치**를 끊김 없이 보여주기 위한 동선. 모바일 웹 기준.
> 업데이트 시점: 화면 흐름 변경/마일스톤 시. 화면 ID는 `docs/05_UI_FLOW.md` 기준(S0~S9).
> 지원 재난유형 3종(폭염·집중호우·감염병), 역할 5종(원장·담임교사·통학버스·조리사/급식·보건담당자).
>
> **사전 준비(데모 직전 필수)**
> - Vercel 배포 URL(https://disastersafety.vercel.app) 또는 로컬 `npm run build && npm start` 확인.
> - 시드 데이터 적재(기관 3~5, 폭염·집중호우·감염병 프로필, 샘플 재난문자). `supabase/seed.sql` 실행.
> - 안전판: 네트워크 불안정 대비 `USE_SAMPLE_FALLBACK=true`로 1회 리허설 완료.
> - 모바일 뷰포트(또는 실제 폰)로 진행.
> - 스모크 테스트 확인: `USE_SAMPLE_FALLBACK=true node scripts/smoke-test.mjs` → **44/44 통과**

## USE_SAMPLE_FALLBACK 무중단 시연(차별화 포인트, 선택 30s)

- 네트워크/AI/DB 차단 상황 가정: `USE_SAMPLE_FALLBACK=true` 또는 키 제거.
- **3개 재난유형 전 흐름이 샘플 결과 배지**와 함께 멈춤 없이 동작함을 시연.
- "장애에도 시연이 멈추지 않는다" — 공모전 데모 안정성 어필.

---

## 안전·개인정보 어필 포인트(심사 강조)

- 입력 화면에 유아 이름/진단명/연락처 **필드 자체가 없음**.
- AI 전송 데이터에 PII 0건(화이트리스트).
- **의료 진단 없음**, 공식기관 우선 표기.
- 감염병: 확진 단정 없음, 익명·집합 표현, 낙인 금지 문구.
- 역할 추천: "배치 기준 확인이 필요합니다" 톤 (법적 단정 금지).

---

## 백업 플랜(데모 사고 대응)

- AI 지연 → 자동 유형별 fallback 결과로 진행(설명 그대로 사용, 샘플 배지 표시).
- DB 오류 → in-memory 샘플 시드로 진행.
- 완전 오프라인 → `USE_SAMPLE_FALLBACK=true` 모드 사전 기동. 3유형 전 흐름 무중단.

---

## P6 공공 API 시연 포인트(선택)

- `/plan/new/message` → **"실시간 조회"** 탭 → "가져오기" 버튼 → 행안부 API 재난문자 목록(`?disaster_type=heavy_rain` 등 유형 필터 가능).
- 기관 등록 주소 입력 → **"자동 채움"** → Kakao Geocode API 좌표 자동 입력.
- AI 결과 `reflected_evidence`에 실측 기온·체감온도 반영(KMA 초단기예보).
- 기상청 폭염영향예보 V2: 지역별 폭염 위험 수준(level) 반영.

# 05_UI_FLOW — 화면 흐름 및 UI 설계

> 스택: Next.js(App Router) + Tailwind + shadcn/ui. 현장 모바일 사용을 가정한 **반응형·버튼형 UI**.
> 모든 결과 화면에 **SafetyNotice 고정 문구**를 노출한다.
> 지원 재난유형: 폭염(heatwave) · 집중호우(heavy_rain) · 감염병(infection) — 3종.
> 지원 역할: 원장 · 담임교사 · 통학버스담당자 · 조리사/급식담당자 · 보건담당자 — 5종.

---

## 1. 화면 목록

| ID | 화면 | 경로(예시) | 핵심 역할 |
|---|---|---|---|
| S0 | 랜딩 | `/` | 서비스 소개 + 로그인/등록 진입 |
| S0a | 기관 로그인 | `/login` | 등록번호 + PIN |
| S1 | 기관 등록 | `/register` | 어린이집포털 API 검색·자동채움 + 로그인정보 설정 |
| S2 | 재난유형별 대응 프로필 | `/institutions/[id]/profile` | 유형별 프로필 탭(집계값), PII 없음. 등록 직후 온보딩 진입 |
| S2c | 역할별 담당자 연락처 | `/account/contacts` | 5역할 이름·휴대폰·이메일 + 수신동의 |
| S3 | 재난문자 입력 | `/plan/new/message` | 실시간 조회 / 원문 붙여넣기 → AI 재난유형 자동분류 |
| S4 | 현재 상황 선택 | `/plan/new/situation` | 버튼형, 최대 3개 + 당일 운영 인원(선택) |
| S5 | AI 대응계획 결과 | `/plan/[requestId]` | 역할별 **읽기형** 대응계획 동적 탭(5역할) + 요약/근거 + 공유 패널 |
| S6 | 학부모 안내문 | S5 내 탭 | 결과 탭 + 공유 링크 |
| S7 | 공유(공개 링크) | `/share/[token]/[role]` | 인증 불필요, 역할별 대응계획 단독 열람 |
| S8 | 관리자 대시보드 | `/admin` | 재난유형별 집계 + 기관 검색·페이지네이션 |
| S9 | 기관 상세 | `/admin/institutions/[id]` | 기관별 최근 대응계획 이력 |

> S3→S4→S5는 하나의 "대응계획 생성 마법사(wizard)"로 묶어 진행 표시(스텝 인디케이터 **2단계: 재난문자 → 상황**)를 둔다.
> 기관은 **로그인 세션**으로 결정되므로 기관 선택 단계가 없다. 재난유형은 **자동 분류**되므로 유형 선택 단계가 없다.
> **제거된 화면**: 역할 선택 랜딩, 기관 선택(`/plan/new`), 재난유형 수동 선택(`/plan/new/type`), 사후기록(`/plan/[id]/after-action`).

## 2. 사용자 흐름 (Happy Path)

```
S0 랜딩 → S0a 기관 로그인
  ├─[미등록] S1 기관 등록(포털 API 자동채움 + 로그인정보) → 자동 로그인
  │     → S2 재난안전 프로필(온보딩) → S2c 담당자 연락처
  ├─[원장(로그인 후)]
  │     → S3 재난문자 입력(실시간 조회/원문) → AI 자동 재난유형 분류
  │     → S4 상황선택(최대3) + 당일 운영 인원
  │     → [생성] → S5 결과(역할 탭 5개 읽기 + 학부모) → 공유 패널(링크/인쇄/발송)
  │     → S7 공유 링크 수신자(담임/통학버스/조리/보건)는 /share/[token]/[role]로 자기 계획 열람
  └─[지자체 관리자]→ S8 대시보드(재난유형별 현황·기관 검색) → S9 기관 상세
```

- **생성 버튼** 클릭 시: 로딩 스켈레톤 표시(`/plan/[requestId]/loading.tsx`) → AI 호출(또는 fallback) → S5 렌더. 실패해도 fallback으로 S5 도달.
- 재난문자 입력(S3) 후 자동 분류된 `disaster_type`을 sessionStorage 기반 WizardDraft에 저장 → S4·생성이 이를 참조. 기관은 세션(`ds_session` 쿠키)으로 결정.

## 3. 마법사(Wizard) 흐름 상세

```
(로그인 세션이 기관 결정 — 기관 선택 단계 없음)
/plan/new/message    (재난문자 입력 — Step 1 of 2) → AI 자동 재난유형 분류
    ↓ 입력·분류 후
/plan/new/situation  (현재상황 선택 — Step 2 of 2) + 당일 운영 인원
    ↓ [대응계획 생성] 버튼 (institution_id는 서버가 세션으로 override)
/plan/[id]           (결과 화면 — 읽기형 + 공유 패널)
```

### 재난문자 입력 화면 (`/plan/new/message`) 상세

- 탭 2개 — [원문 붙여넣기] / [실시간 조회(행안부 API)]. **화면상 샘플 선택 탭 제거.**
- 진입 시 이전 WizardDraft 초기화(새 대응계획은 항상 새로 시작).
- 입력 후 "다음" → `POST /api/plan/classify`로 재난유형 자동 분류:
  - 분류 성공(폭염/집중호우/감염병) → 토스트로 유형 안내 후 상황 단계 이동.
  - 실시간 조회 항목이 이미 분류돼 있으면 그 값 사용.
  - 분류 실패('other') → 3유형 수동 선택 UI 노출(폴백).
- 감염병처럼 공식 재난문자가 없는 경우에도 원문 붙여넣기에 기관 상황을 적어 진행 가능(빈 문자 허용).

## 4. 역할별 화면

| 역할 | 진입/주력 화면 | 강조 요소 |
|---|---|---|
| 지자체 관리자 | S8 대시보드 → S9 | 재난유형별 건수·오늘 생성·우선순위 高·활성기관, 기관 검색·페이지네이션 |
| 원장(로그인) | S3~S5 전체 + 공유 패널 | 재난문자 입력·생성, 역할별 공유/발송 총괄 |
| 담임교사 | S7 공유 링크(`/share/[token]/homeroom_teacher`) | 담임 대응계획 단독 열람 |
| 통학버스 담당자 | S7 공유 링크(`.../bus_manager`) | 통학버스 대응계획, 탑승 전/하차 점검 |
| 조리사/급식담당자 | S7 공유 링크(`.../cook_or_food_service`) | 급식·조리실 대응계획, 식재료·위생 |
| 보건담당자 | S7 공유 링크(`.../health_manager`) | 증상 관찰·분리대기·응급 연락 기준 |

- 상단 헤더에 로그인 기관명 배지 + 로그아웃. 비로그인 역할 담당자는 원장이 공유한 링크/문자로만 접근(별도 로그인 불필요).

## 5. 핵심 컴포넌트

| 컴포넌트 | 역할 | 비고 |
|---|---|---|
| `SessionProvider` | 로그인 세션 컨텍스트 | `/api/auth/session` 조회, 헤더 기관명·로그아웃 |
| (로그인 폼) | 기관 로그인(S0a) | 등록번호 + PIN |
| (등록 폼) | 기관 등록(S1) | 어린이집포털 API 검색→자동채움 + 로그인정보 설정. `handleGeocode` 재사용 |
| `ProfileForm` + `ProfileTypeTabs` | 재난유형별 프로필(S2) | 재난유형별 탭 전환. 유형별 입력 필드. 숫자 스텝퍼·토글 위주, PII 없음 |
| `StaffContactsForm` | 담당자 연락처(S2c) | 5역할 이름·휴대폰·이메일 + 수신동의 토글 |
| `MessageInput` | 재난문자(S3) | 탭: [원문 붙여넣기]/[실시간 조회]. 입력→자동분류, 분류 실패 시 수동 선택 폴백 |
| `SituationPicker` | 상황 선택(S4) | `DISASTER_REGISTRY` 동적 로드. 토글 **최대 3개** + 당일 운영 인원 입력 |
| `PlanResult` | 결과(S5) | `role_based_actions` 기반 역할 탭(5역할+학부모) **읽기 전용** 렌더. 요약·우선순위·근거·부족정보 + 공유 패널 |
| `ParentNoticeCard` | 학부모 안내문(S6) | 본문 + 복사 버튼(클립보드) |
| `SharePanel` | 공유(S5 하단) | 역할별 링크 복사 / 출력·PDF(`window.print`) / 발송(SMS·알림톡) |
| (공유 페이지) | 공개 열람(S7) | `/share/[token]/[role]` 인증 불필요 읽기 + SafetyNotice |
| `AdminDashboard` + `DisasterTypeSummary` + `InstitutionTable` | 대시보드(S8) | 재난유형별 건수 + 검색·페이지네이션 기관 목록 |
| `SafetyNotice` | 안전 고지 | 결과·안내문·공유 화면 고정 노출 |
| `FallbackBadge` | 샘플/실데이터 표시 | `is_fallback` 기반 |

### S5(PlanResult) 레이아웃 개요

```
[우선순위 배지: 높음]   [샘플 결과 배지(있을 때)]
재난문자/상황 요약 ............................
반영된 근거 정보 (리스트)
부족한 정보 (리스트)
────────────────────────────
[탭: 원장 | 담임교사 | 통학버스 | 조리사/급식 | 보건담당자 | 학부모 안내문]
  ← role_based_actions 기반 동적 생성 (5역할 + 학부모 탭)
  - 각 역할 탭: 읽기 전용 번호 목록(체크박스·토글 없음)
  - 학부모 탭: ParentNoticeCard(복사)
  - 해당 인력 없는 역할: "해당 없음" 메시지 표시
응급 연락 안내 .........
공식기관 지시 우선 안내문 .........
[SafetyNotice 고정 문구]
[SharePanel: 역할별 링크 복사 / 출력·PDF / 담당자 발송]
```

### S2(ProfileForm + ProfileTypeTabs) 레이아웃 개요

```
[탭: 폭염 | 집중호우 | 감염병]
  - 폭염: 온열취약 유아 수, 냉방 여부, 수분 공급, 차량 온도계, 하원 대기 장소 등
  - 집중호우: 저지대 여부, 인근 하천·급경사지, 지하공간, 1층 출입구 구조, 대피 가능 공간 등
  - 감염병: 반별 유아수(집계), 영아반 여부, 보건실/분리대기 공간, 체온계, 소독용품, 보호자 연락 방식 등
```

### S8(관리자 대시보드) 레이아웃 개요

```
[통계 카드: 등록 기관 수 / 오늘 생성 수 / 우선순위 高] + 최근 30일 활성 기관 수
DisasterTypeSummary: [폭염 N건] [집중호우 N건] [감염병 N건]
[최근 대응계획 목록 — 재난유형 뱃지 포함]
[기관 목록 — 검색 + 페이지네이션(전체 로드 폐지, /api/admin/institutions)]
```

## 6. 모바일 웹 고려사항

- 현장에서 스마트폰으로 사용 → **세로 단일 컬럼**, 큰 터치 타깃(최소 44~48px), 하단 고정 주요 액션 버튼.
- 상황 선택은 칩/토글 버튼 그리드, 최대 3개 초과 시 비활성·토스트 안내.
- 결과 역할 탭은 읽기 전용 번호 목록(공유 대상에게 그대로 전달 가능한 형태).
- 학부모 안내문 복사·공유 링크 복사는 눈에 띄게(`navigator.clipboard` — 복사 완료 토스트).
- 출력/PDF는 `@media print`로 네비·버튼·공유 패널 숨기고 본문만 인쇄.
- 로딩/에러 상태 명확화(스켈레톤, fallback 안내 배너).
- 입력 폼은 숫자 키패드 유도(`inputMode="numeric"`), PIN은 숫자만.

## 7. 상태·전환 메모

- 대응계획 생성 중: 로딩 스켈레톤(`/plan/[requestId]/loading.tsx`) + 취소 불가(짧게). 타임아웃 시 fallback 자동.
- 미인증 상태로 보호경로 접근: `proxy.ts`가 `/login`으로 redirect(`?next=`).
- 접근성: 색상만으로 우선순위 구분하지 않고 텍스트 라벨 병기.
- 재난문자 입력 단계 진입 시 WizardDraft 초기화(이전 생성 잔여 제거).
- 공유 링크(`/share/[token]/[role]`)는 token 비밀성에 의존하는 공개 페이지 — 발송 본문/페이지에 유아 PII 미포함.

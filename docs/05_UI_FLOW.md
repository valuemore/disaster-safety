# 05_UI_FLOW — 화면 흐름 및 UI 설계

> 스택: Next.js(App Router) + Tailwind + shadcn/ui. 현장 모바일 사용을 가정한 **반응형·버튼형 UI**.
> 모든 결과 화면에 **SafetyNotice 고정 문구**를 노출한다.
> 지원 재난유형: 폭염(heatwave) · 집중호우(heavy_rain) · 감염병(infection) — 3종.
> 지원 역할: 원장 · 담임교사 · 통학버스담당자 · 조리사/급식담당자 · 보건담당자 — 5종.

---

## 1. 화면 목록

| ID | 화면 | 경로(예시) | 핵심 역할 |
|---|---|---|---|
| S0 | 랜딩 / 역할 선택 | `/` | 4개 역할 선택(시연용) |
| S1 | 기관 등록 | `/institutions/new` | 기관 정보 입력(급식·보건 인력 포함) |
| S2 | 재난유형별 대응 프로필 | `/institutions/[id]/profile` | 유형별 프로필 탭(집계값), PII 없음 |
| **S2a** | **재난유형 선택** | **`/plan/new/type`** | **3종 재난유형 카드 선택(신규 화면)** |
| S3 | 재난문자 입력 | `/plan/new/message` | 샘플 선택/원문 붙여넣기/실시간 조회(감염병: 2-모드 토글) |
| S4 | 현재 상황 선택 | `/plan/new/situation` | 버튼형, 최대 3개, 재난유형별 상황 버튼 |
| S5 | AI 대응계획 결과 | `/plan/[requestId]` | 역할별 체크리스트 동적 탭(5역할) + 요약/근거 |
| S6 | 학부모 안내문 | S5 내 섹션/탭 | 복사 버튼 |
| S7 | 사후기록 | `/plan/[requestId]/after-action` | 재난유형별 checked_items, AI 초안 자동 채움 |
| S8 | 관리자 대시보드 | `/admin` | 재난유형별 집계·목록, 기관 역할 지정 현황 |
| S9 | 기관 상세 | `/admin/institutions/[id]` | 기관별 최근 대응계획 이력 |

> S2a→S3→S4→S5는 하나의 "대응계획 생성 마법사(wizard)"로 묶어 진행 표시(스텝 인디케이터 4단계)를 둔다.
> 기관·프로필이 이미 있으면 S1·S2를 건너뛰고 S2a(기관 선택)부터 시작 가능.

## 2. 사용자 흐름 (Happy Path)

```
S0 역할 선택
  ├─[원장/담임/통학버스/조리사/보건]→ (기관 선택 또는 S1 등록)→ S2 프로필 확인/등록
  │     → S2a 재난유형 선택(폭염·집중호우·감염병)
  │     → S3 재난문자/상황 입력 → S4 상황선택(최대3)
  │     → [생성] → S5 결과(역할 탭 5개) → S6 학부모 안내문 복사 → S7 사후기록 저장 → 완료
  └─[지자체 관리자]→ S8 대시보드(재난유형별 현황) → S9 기관 상세 → (필요 시 S5 결과 열람)
```

- **생성 버튼** 클릭 시: 로딩 스켈레톤 표시(`/plan/[requestId]/loading.tsx`) → AI 호출(또는 fallback) → S5 렌더. 실패해도 fallback으로 S5 도달.
- 재난유형 선택(S2a) 후 sessionStorage 기반 WizardDraft에 `disaster_type` 저장 → 이후 모든 화면이 이를 참조.

## 3. 마법사(Wizard) 흐름 상세

```
/plan/new            (기관 선택 — Step 1 of 4)
    ↓ 기관 선택 후
/plan/new/type       (재난유형 선택 — Step 2 of 4) ← 신규
    ↓ 유형 카드 클릭 후
/plan/new/message    (재난문자 입력 — Step 3 of 4)
    ↓ 선택/입력 후
/plan/new/situation  (현재상황 선택 — Step 4 of 4)
    ↓ [대응계획 생성] 버튼
/plan/[id]           (결과 화면)
```

### 재난유형 선택 화면 (`/plan/new/type`) 상세

- DISASTER_REGISTRY 순회 → 3종 카드 렌더링.
- 폭염·집중호우·감염병 모두 활성(`enabled:true`). 카드 클릭 → `WizardDraft.disaster_type` 저장 후 `/plan/new/message` 이동.
- 딥링크 직접 진입 시 `institution_id` 없으면 `/plan/new` 리다이렉트(가드).

### 재난문자 입력 화면 (`/plan/new/message`) 상세

- **폭염·집중호우**: 탭 3개 — [샘플 선택] / [원문 붙여넣기] / [실시간 조회(행안부 API)].
  - 재난유형별 샘플 메시지 목록(`getSampleMessagesByType`) 자동 분기.
  - 실시간 조회 시 `disaster_type` 쿼리 파라미터 전달 → 유형별 필터.
- **감염병**: 2-모드 토글.
  - **모드 A (보건당국 안내문자)**: 보건당국 안내 문자 붙여넣기 또는 샘플 선택.
  - **모드 B (기관 내 상황 입력)**: 재난문자 없이 기관 내 발생 상황 입력만으로 진행. `disaster_message_text`는 비워도 허용.

## 4. 역할별 화면

| 역할 | 진입/주력 화면 | 강조 요소 |
|---|---|---|
| 지자체 관리자 | S8 대시보드 → S9 | 재난유형별 등록 기관 수·오늘 생성 수·우선순위 高 기관, 역할 지정 현황 |
| 원장 | S5(원장 탭) + 생성 전체 | 원장 체크리스트, 안내문 승인, 전체 조망 |
| 담임교사 | S5(담임교사 탭) | 담임 체크리스트, 체크 토글 |
| 통학버스 담당자 | S5(통학버스 탭) | 통학버스 체크리스트, 탑승 전/하차 점검 |
| 조리사/급식담당자 | S5(조리사/급식 탭) | 급식·조리실 관련 체크리스트, 식재료·위생 |
| 보건담당자 | S5(보건담당자 탭) | 증상 관찰·분리대기·응급 연락 기준 |

- 역할은 시연용 선택이므로, 상단에 현재 역할 배지 + 역할 전환 버튼 제공.

## 5. 핵심 컴포넌트

| 컴포넌트 | 역할 | 비고 |
|---|---|---|
| `RoleSelector` | 역할 선택(S0) | 큰 버튼/카드 |
| `InstitutionForm` | 기관 등록(S1) | 주소→좌표 자동(Kakao API)·실패 시 수기. 급식·보건 인력 입력 섹션 포함 |
| `ProfileForm` + `ProfileTypeTabs` | 재난유형별 프로필(S2) | 재난유형별 탭 전환. 유형별 다른 입력 필드(폭염/집중호우/감염병). 숫자 스텝퍼·토글 위주, PII 없음 |
| `MessageInput` | 재난문자(S3) | 탭: [샘플 선택]/[원문 붙여넣기]/[실시간 조회]. 감염병 2-모드 토글. 재난유형별 샘플 자동 분기 |
| `SituationPicker` | 상황 선택(S4) | `DISASTER_REGISTRY`에서 재난유형별 situations 동적 로드. 토글 버튼, **최대 3개** 제한, '기타' 입력 |
| `PlanResult` | 결과(S5) | `role_based_actions` 기반 역할 탭 동적 생성(5역할+학부모). 역할 추천 패널. 요약·우선순위 배지·근거·부족정보 |
| `ChecklistCard` | 체크리스트 | 항목 체크 토글, 진행률. cook_or_food_service/health_manager 포함 5역할 |
| `ParentNoticeCard` | 학부모 안내문(S6) | 본문 + **복사 버튼**(클립보드) |
| `AfterActionForm` | 사후기록(S7) | `checked_items` JSONB 기반, AI 초안 자동 채움 후 수정 |
| `AdminDashboard` + `DisasterTypeSummary` | 대시보드(S8) | 재난유형별 건수 뱃지·막대. 기관 역할 지정 현황 |
| `SafetyNotice` | 안전 고지 | 결과·안내문 화면 고정 노출 |
| `FallbackBadge` | 샘플/실데이터 표시 | `is_fallback`/`source` 기반 |

### S5(PlanResult) 레이아웃 개요

```
[우선순위 배지: 높음]   [샘플 결과 배지(있을 때)]
재난문자/상황 요약 ............................
반영된 근거 정보 (리스트)
부족한 정보 (리스트)
────────────────────────────
[탭: 원장 | 담임교사 | 통학버스 | 조리사/급식 | 보건담당자 | 학부모 안내문]
  ← role_based_actions 기반 동적 생성 (5역할 + 학부모 탭)
  - 각 탭: ChecklistCard / ParentNoticeCard
  - 해당 인력 없는 역할: "해당 없음" 메시지 표시
역할 추천 패널 (기관 규모·유형 기반, "확인이 필요합니다" 톤)
응급 연락 안내 .........
공식기관 지시 우선 안내문 .........
[SafetyNotice 고정 문구]
[버튼: 사후기록 작성하기]
```

### S2(ProfileForm + ProfileTypeTabs) 레이아웃 개요

```
[탭: 폭염 | 집중호우 | 감염병]
  - 폭염: 온열취약 유아 수, 냉방 여부, 수분 공급, 차량 온도계, 하원 대기 장소 등
  - 집중호우: 저지대 여부, 인근 하천·급경사지, 지하공간, 1층 출입구 구조, 대피 가능 공간 등
  - 감염병: 반별 유아수(집계), 영아반 여부, 보건실/분리대기 공간, 체온계, 소독용품, 보호자 연락 방식 등
```

### S8(관리자 대시보드) 레이아웃 개요

```
[통계 카드: 등록 기관 수 / 오늘 생성 수 / 우선순위 高]
DisasterTypeSummary: [폭염 N건] [집중호우 N건] [감염병 N건]
[최근 대응계획 목록 — 재난유형 뱃지 포함]

```

## 6. 모바일 웹 고려사항

- 현장에서 스마트폰으로 사용 → **세로 단일 컬럼**, 큰 터치 타깃(최소 44~48px), 하단 고정 주요 액션 버튼.
- 상황 선택은 칩/토글 버튼 그리드, 최대 3개 초과 시 비활성·토스트 안내.
- 체크리스트는 탭 한 번으로 완료 토글, 진행률 시각화.
- 학부모 안내문 복사 버튼은 눈에 띄게(`navigator.clipboard` — 복사 완료 토스트).
- 로딩/에러 상태 명확화(스켈레톤, fallback 안내 배너).
- 입력 폼은 숫자 키패드 유도(`inputMode="numeric"`), 토글/스텝퍼로 오타 최소화.
- 재난유형 선택 카드: 큰 아이콘 + 유형명 + 설명, 비활성 카드는 `opacity-40`·클릭 차단(현재 전부 활성).

## 7. 상태·전환 메모

- 대응계획 생성 중: 로딩 스켈레톤(`/plan/[requestId]/loading.tsx`) + 취소 불가(짧게). 타임아웃 시 fallback 자동.
- 빈 상태(기관 없음 등): 안내 + 샘플 데이터로 시작 유도(데모 친화).
- 접근성: 색상만으로 우선순위 구분하지 않고 텍스트 라벨 병기.
- 재난유형 선택 후 새로고침 시 sessionStorage 기반 WizardDraft 초기화 가능 — 시연 중 새로고침 자제 안내 필요.
- 감염병 생성 시 `disaster_message_id: null` 허용 — 재난문자 없이 상황만으로 400 발생하지 않음.

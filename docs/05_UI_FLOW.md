# 05_UI_FLOW — 화면 흐름 및 UI 설계

> 스택: Next.js(App Router) + Tailwind + shadcn/ui. 현장 모바일 사용을 가정한 **반응형·버튼형 UI**.
> 모든 결과 화면에 **SafetyNotice 고정 문구**를 노출한다.

---

## 1. 화면 목록

| ID | 화면 | 경로(예시) | 핵심 역할 |
|---|---|---|---|
| S0 | 랜딩 / 역할 선택 | `/` | 4개 역할 선택(시연용) |
| S1 | 기관 등록 | `/institutions/new` | 6.2 입력 |
| S2 | 폭염 대응 프로필 | `/institutions/[id]/profile` | 6.3 입력(집계값) |
| S3 | 재난문자 입력 | `/plan/new/message` | 샘플 선택/원문 붙여넣기 |
| S4 | 현재 상황 선택 | `/plan/new/situation` | 버튼형, 최대 3개 |
| S5 | AI 대응계획 결과 | `/plan/[requestId]` | 역할별 체크리스트 탭 + 요약/근거 |
| S6 | 학부모 안내문 | S5 내 섹션/탭 | 복사 버튼 |
| S7 | 사후기록 | `/plan/[requestId]/after-action` | 6.8 입력·저장 |
| S8 | 관리자 대시보드 | `/admin` | 6.9 집계·목록 |
| S9 | 기관 상세 | `/admin/institutions/[id]` | 기관별 최근 대응계획 |

> S3→S4→S5는 하나의 "대응계획 생성 마법사(wizard)"로 묶어 진행 표시(스텝 인디케이터)를 둔다.

## 2. 사용자 흐름 (Happy Path)

```
S0 역할 선택
  ├─[원장/담임/통학버스]→ (기관 선택 또는 S1 등록)→ S2 프로필 확인/등록
  │     → S3 재난문자 → S4 상황선택(최대3) → [생성]
  │     → S5 결과(역할 탭) → S6 학부모 안내문 복사 → S7 사후기록 저장 → 완료
  └─[지자체 관리자]→ S8 대시보드 → S9 기관 상세 → (필요 시 S5 결과 열람)
```

- **생성 버튼** 클릭 시: 로딩 표시 → AI 호출(또는 fallback) → S5 렌더. 실패해도 fallback으로 S5 도달.
- 기관·프로필이 이미 있으면 S1·S2를 건너뛰고 S3부터 시작 가능.

---

## 3. 역할별 화면

| 역할 | 진입/주력 화면 | 강조 요소 |
|---|---|---|
| 지자체 관리자 | S8 대시보드 → S9 | 등록 기관 수, 오늘 생성 수, 우선순위 高 기관, 기관별 최근 계획 |
| 원장 | S5(원장 탭) + 생성 전체 | 원장 체크리스트, 안내문 승인, 전체 조망 |
| 담임교사 | S5(담임 탭) | 담임 체크리스트, 체크 토글 |
| 통학버스 담당자 | S5(통학버스 탭) | 통학버스 체크리스트, 탑승 전/하차 점검 |

- 역할은 시연용 선택이므로, 상단에 현재 역할 배지 + 역할 전환 버튼 제공.

## 4. 핵심 컴포넌트

| 컴포넌트 | 역할 | 비고 |
|---|---|---|
| `RoleSelector` | 역할 선택(S0) | 4개 큰 버튼/카드 |
| `InstitutionForm` | 기관 등록(S1) | 주소→좌표 자동(03 API)·실패 시 수기 |
| `ProfileForm` | 폭염 프로필(S2) | 숫자 스텝퍼·토글 위주, PII 입력 필드 없음 |
| `MessageInput` | 재난문자(S3) | 탭: [샘플 선택]/[원문 붙여넣기], (선택)최근 문자 불러오기 |
| `SituationPicker` | 상황 선택(S4) | 토글 버튼, **최대 3개** 제한, '기타' 입력 |
| `PlanResult` | 결과(S5) | 요약·우선순위 배지·근거·부족정보 + 역할 탭 |
| `ChecklistCard` | 체크리스트 | 항목 체크 토글, 진행률 |
| `ParentNoticeCard` | 학부모 안내문(S6) | 본문 + **복사 버튼**(클립보드) |
| `AfterActionForm` | 사후기록(S7) | 6.8 항목, AI 초안 자동 채움 후 수정 |
| `AdminDashboard` | 대시보드(S8) | 통계 카드 + 목록 테이블 |
| `SafetyNotice` | 안전 고지 | 결과·안내문 화면 고정 노출 |
| `FallbackBadge` | 샘플/실데이터 표시 | `is_fallback`/`source` 기반 |

### S5(PlanResult) 레이아웃 개요
```
[우선순위 배지: 높음]   [샘플 결과 배지(있을 때)]
재난문자 요약 ............................
반영된 근거 정보 (리스트)
부족한 정보 (리스트)
────────────────────────────
[탭: 원장 | 담임교사 | 통학버스 | 학부모 안내문]
  - 각 탭: ChecklistCard / ParentNoticeCard
응급 연락 안내 .........
공식기관 지시 우선 안내문 .........
[SafetyNotice 고정 문구]
[버튼: 사후기록 작성하기]
```

## 5. 모바일 웹 고려사항
- 현장에서 스마트폰으로 사용 → **세로 단일 컬럼**, 큰 터치 타깃(최소 44px), 하단 고정 주요 액션 버튼.
- 상황 선택은 칩/토글 버튼 그리드, 최대 3개 초과 시 비활성·토스트 안내.
- 체크리스트는 탭 한 번으로 완료 토글, 진행률 시각화.
- 학부모 안내문 복사 버튼은 눈에 띄게(복사 완료 토스트).
- 로딩/에러 상태 명확화(스켈레톤, fallback 안내 배너).
- 입력 폼은 숫자 키패드 유도(`inputMode="numeric"`), 토글/스텝퍼로 오타 최소화.

---

## 6. 상태·전환 메모
- 대응계획 생성 중: 로딩 오버레이 + 취소 불가(짧게). 타임아웃 시 fallback 자동.
- 빈 상태(기관 없음 등): 안내 + 샘플 데이터로 시작 유도(데모 친화).
- 접근성: 색상만으로 우선순위 구분하지 않고 텍스트 라벨 병기.

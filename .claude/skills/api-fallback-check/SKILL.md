# api-fallback-check — 공공 API fallback 검증

기준: `docs/03_API_INTEGRATION_PLAN.md`.

## 언제 호출
- 외부 공공 API route(`app/api/external/**`) 또는 연동 클라이언트를 추가/변경할 때.

## 공통 점검
- [ ] 외부 호출은 **Next.js Server Route에서만** 수행(클라이언트 직접 호출 없음).
- [ ] API 키/`service_role`은 서버 환경변수에서만 사용, 클라이언트 노출 없음.
- [ ] 응답에 `source: 'api' | 'sample'` 메타 포함.

## Fallback 분기 점검
- [ ] `USE_SAMPLE_FALLBACK=true` → 외부 호출 시도 없이 즉시 샘플 반환.
- [ ] **키 미설정** → 자동 샘플 모드(빌드/기동 실패 없음).
- [ ] **타임아웃(기본 5s)** → 1회 재시도 → 실패 시 샘플 반환.
- [ ] **에러 응답(4xx/5xx)** → 샘플 반환, 흐름 무중단.
- [ ] 어떤 외부 API 없이도 **핵심 흐름(AI 생성·결과·사후기록) 100% 동작**.

## 테스트 절차
1. 키 제거 후 기동 → 기관등록/예보 등에서 샘플 응답 확인.
2. 잘못된 키/네트워크 차단 → 재시도 후 샘플 전환 확인.
3. `USE_SAMPLE_FALLBACK` 토글 양방향 확인.
4. UI에 실데이터/샘플 배지 표시 확인.

## 대상 API(우선순위)
- 주소·좌표(高) → 기상청 단기예보(高) → 영향예보(中) → 행안부 재난문자(中) → 어린이집/유치원 정보(低). 각 fallback은 `docs/03` 표 참조.

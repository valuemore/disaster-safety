# privacy-safety-review — 개인정보·안전 표현 점검

기준: `CLAUDE.md` 개인정보/AI 안전 원칙, `docs/00_PRD.md` §6·10, `docs/02_DB_SCHEMA.md`.

## 언제 호출
- 폼/저장/프롬프트/사용자 노출 문구 변경 직후. 제출 전 일괄 점검.

## 개인정보 점검
- [ ] 유아 **이름·진단명·약물명·보호자 연락처** 입력 필드/컬럼/변수 **없음**.
- [ ] 취약 유아 정보는 **숫자 집계값**만(boolean/int). 자유텍스트에 PII 입력 유도 없음.
- [ ] DB 스키마에 PII 컬럼 부재.
- [ ] **AI API 전송 페이로드에 PII 0건**(화이트리스트 직렬화 확인).
- [ ] 외부 공공 API에 비식별 정보(좌표·지역코드)만 전송.
- [ ] 사후기록 자유텍스트(특이사항 등)에 PII 입력 자제 안내 존재.

## 안전 표현 점검
- [ ] AI/UI가 **의료 진단·질병 단정·약물 권고**를 하지 않음.
- [ ] **공식기관 지시보다 우선한다는 표현 없음**. 항상 공식기관·119 우선 안내.
- [ ] "AI가 위험을 예측" 단정 대신 **"대응 우선순위 제안"** 표현.
- [ ] 결과/안내 화면에 고정 안전 문구 노출: "공식 재난문자와 기관 입력정보를 바탕으로 한 대응지원 정보이며, 위급상황에서는 공식기관 지시와 119 안내를 우선합니다."
- [ ] 학부모 안내문: 과도한 공포 표현 없음, 조치 중 안정감 + 구체 행동.
- [ ] 허위 연락처/기관명 등 생성 없음.

## 산출
- 위반 발견 시 위치와 수정안을 명시(이 스킬은 점검 중심). 수정은 구현 주체가 반영.

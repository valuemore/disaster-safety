# demo-readiness — 시연 준비 점검

기준: `docs/09_DEMO_SCRIPT.md`, `docs/10_HARNESS_PLAN.md`.

## 언제 호출
- 마일스톤(M1~M4) 도달 시, 공모전 제출 직전.

## 3분 시연 점검
- [ ] `docs/09_DEMO_SCRIPT.md` 메인 시나리오 S0→S7 **완주**(3분 내).
- [ ] 입력→결과 연결(재난문자 요약·반영 근거)이 화면에 드러남.
- [ ] 역할별 체크리스트 탭 차별화 노출.
- [ ] 학부모 안내문 복사 버튼 동작.
- [ ] 관리자 대시보드 우선순위 식별.

## 샘플/장애 내성 점검
- [ ] 시드 데이터 적재 상태 확인.
- [ ] `USE_SAMPLE_FALLBACK=true`로 외부 의존 차단 시 전 흐름 동작.
- [ ] AI/공공 API/DB 실패 주입 시 샘플 fallback + `is_fallback` 배지로 무중단.

## 모바일 점검
- [ ] 세로 단일 컬럼, 터치 타깃 충분, 버튼형 상황 선택.
- [ ] 로딩/에러/스켈레톤 상태 명확.

## 안전 점검
- [ ] 안전 고지 문구 노출, PII 미수집(→ `/privacy-safety-review`).

## 마무리
- [ ] 통과 시 `docs/07_CONTEXT_LEDGER.md` "마지막 정상 시연" 갱신(날짜·경로·커밋).
- [ ] 미해결 항목은 리스크로 Ledger에 기록.
